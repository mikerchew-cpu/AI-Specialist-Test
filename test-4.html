TUGAS 1: MERANGKUM TRANSKRIP RAPAT YANG BERANTAKAN TASK 1: SUMMARIZING MESSY MEETING MINUTES
Skenario Kantor:Office Scenario:
Anda memiliki teks catatan rapat internal sepanjang 3 halaman yang kalimatnya berantakan, lompat-lompat, dan dipenuhi obrolan kosong. Manajer Anda meminta ringkasan eksekutif instan yang fokus pada Action Items (Siapa harus melakukan apa, dan apa tenggat waktunya). You hold a 3-page unedited transcript of internal meeting notes that is highly unstructured and scattered. Your manager orders an instant executive summary focusing strictly on Action Items (Who must do what, and the specific deadlines).
❌ Contoh Prompt Buruk (Boros & Tidak Jelas):Bad Prompt Example (Wasteful & Vague): "Rangkum teks ini ya." / "Summarize this text."
📖 Lihat Contoh Jawaban / View Sample Answer
Contoh Jawaban:Sample Answer:
"Bertindaklah sebagai Sekretaris Korporat Profesional. Rangkum teks transkrip rapat di bawah ini menjadi Memo Ringkas. Abaikan obrolan basa-basi. Format keluaran wajib berupa tabel dengan kolom: No, Nama Karyawan, Tugas yang Ditugaskan, dan Tenggat Waktu." "Act as an Expert Corporate Secretary. Distill the attached unstructured notes into an Executive Memo. Filter out casual side talks. Format output into a table with columns: No, Owner Name, Assigned Action Item, and Expected Deadline."
💡 Kerangka Struktur Jawaban Ideal:Ideal Response Blueprint Framework:
Prompt Anda harus mengandung unsur: "Bertindaklah sebagai Sekretaris Korporat Profesional [Persona]. Rangkum teks transkrip rapat di bawah ini menjadi Memo Ringkas [Tugas]. Abaikan obrolan basa-basi [Batasan]. Format keluaran wajib berupa tabel yang berisi kolom: No, Nama Karyawan, Tugas yang Ditugaskan, dan Tenggat Waktu [Format Kendali]."
Your prompt block should incorporate elements of: "Act as an Expert Corporate Secretary [Persona]. Distill the attached unstructured notes into an Executive Memo [Task]. Filter out casual side talks [Negative Constraint]. Format output into a markdown table with columns: No, Owner Name, Assigned Action Item, and Expected Deadline [Output Format Customization]."
TUGAS 2: MENYARING DATA DATA KELUHAN PELANGGAN SECARA TERSTRUKTUR TASK 2: FILTERING CUSTOMER COMPLAINT DATA INTO STRUCTURED FORMATS
Skenario Kantor:Office Scenario:
Divisi operasional menerima puluhan email keluhan pelanggan setiap hari. Anda ditugaskan meminta AI menyaring email-email tersebut secara massal guna mendeteksi email mana yang bernada Marah (Urgent) dan apa jenis produk yang dikeluhkan, agar bisa langsung dimasukkan ke database Excel kantor. The operations team receives dozens of raw consumer grievance emails daily. You are tasked with instructing an AI to batch-scan these entries to extract Sentiment Tier (Urgent/Normal) and the specific Product Category involved for Microsoft Excel ingestion logs.
❌ Contoh Prompt Buruk:Bad Prompt Example: "Tolong baca email komplain keluhan keluhan ini dan kelompokkan." / "Please read these customer issues and categorize them."
📖 Lihat Contoh Jawaban / View Sample Answer
Contoh Jawaban:Sample Answer:
"Bertindaklah sebagai Data Specialist QC. Analisis teks email keluhan di dalam tanda ###. Ekstrak informasi dalam format JSON dengan kunci: 'customer_name', 'sentiment_level', 'product_type'. Jangan berikan teks pembuka atau penutup, langsung keluarkan JSON." "Act as a Data Quality Analyst. Evaluate the customer email wrapped between ###. Extract into strict JSON with keys: 'customer_name', 'sentiment_level', 'product_type'. Omit any conversational preamble."
💡 Kerangka Struktur Jawaban Ideal:Ideal Response Blueprint Framework:
Prompt Anda harus mengandung unsur pembatas data yang jelas: "Bertindaklah sebagai Data Specialist QC. Analisis teks email keluhan di dalam tanda pagar tiga (###). Ekstrak informasi penting dalam format JSON terstruktur yang berisi variabel kunci: 'customer_name', 'sentiment_level', dan 'product_type'. Jangan berikan teks pembuka atau penutup, langsung keluarkan format JSON objek saja."
Your prompt string should encompass strict delimiter isolation properties: "Act as a Data Quality Analyst. Evaluate the customer email input wrapped between triple hashtags (###). Extract indices into a rigid structured JSON array template containing keys: 'customer_name', 'sentiment_level', and 'product_type'. Omit any conversational preambles or chat explanations."
TUGAS 3: REKAYASA PROMPT HEMAT BIAYA (TOKEN MANAGEMENT) TASK 3: ECONOMICAL PROMPT ENGINEERING (TOKEN BURN MITIGATION)
Skenario Kantor:Office Scenario:
Perusahaan melanggan API berbayar (seperti OpenAI/Claude) di mana setiap kata yang dikirim dihitung biayanya (token cost). Staf admin sering melakukan kesalahan dengan menyalin ulang (*copy-paste*) instruksi panduan SOP kantor sepanjang 5 halaman berulang-ulang di setiap baris pertanyaan chat baru, membuat tagihan bulanan kantor jebol. The company pays on-demand usage metrics for model API integrations where every word wire-transmitted levies variable expenses. Admin staff continuously copy-paste full 5-page internal SOP documents over and over inside every single new chat box line question, driving massive corporate billing leaks.
❌ Contoh Prompt Buruk:Bad Prompt Example: [Menempel isi 5 halaman SOP dokumen penuh] + "Berdasarkan teks di atas, apakah cuti melahirkan dibayar?" / [Pastes full 5-page context sheet] + "Based on text above, is maternity leave fully compensated?"
📖 Lihat Contoh Jawaban / View Sample Answer
Contoh Jawaban:Sample Answer:
"1) Gunakan Claude Projects / Custom GPT untuk menyimpan file SOP 5 halaman sekali di system prompt. 2) Terapkan negative prompting agar output AI singkat dan padat, menghemat token keluaran." "1) Deploy Claude Projects / Custom GPT to store the 5-page SOP once in the system prompt. 2) Apply negative constraints limiting output length to reduce token costs."
💡 Kerangka Strategi Finansial Ideal:Ideal Financial Strategy Blueprint:
Kandat harus menjawab dengan solusi fungsional taktis: 1) Memanfaatkan fitur **Claude Projects (Knowledge Base)** atau **Custom GPTs** untuk menaruh file SOP 5 halaman itu satu kali saja di tingkat sistem, sehingga karyawan cukup mengetik pertanyaan pendek tanpa melampirkan teks berulang-ulang. 2) Menggunakan instruksi batasan ketat (*Negative Prompting*) agar output jawaban AI pendek dan padat, menghemat token keluaran (*output token expenses*).
The employee should answer with operational workflow tweaks: 1) Deploying the **Claude Projects (Knowledge Base)** or **Custom GPTs** configurations to store the 5-page master asset exactly once inside system layers, permitting staff to submit light textual queries without attaching context blocks repeatedly. 2) Locking strict *Negative Constraints* limiting output string lengths to prune down output token tracking bills.
